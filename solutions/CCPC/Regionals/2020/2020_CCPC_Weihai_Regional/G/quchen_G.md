### 题目：G-Caesar Cipher

## 题意概述

给定一个长度为 `n` 的数组 `a`，其中每个元素满足：

`0 <= a[i] < 65536`

现在有 `q` 次操作，操作分为两种。

第一种操作：

选择一个区间 `[l, r]`，对区间内所有元素执行：

`a[i] = (a[i] + 1) mod 65536`

第二种操作：

给定 `x, y, len`，询问下面两个长度为 `len` 的子数组是否完全相同：

`a[x], a[x + 1], ..., a[x + len - 1]`

和

`a[y], a[y + 1], ..., a[y + len - 1]`

如果相同输出 `yes`，否则输出 `no`。

---

## 关键观察

这题如果没有 `mod 65536`，那么区间加一非常好处理。

我们可以维护数组的多项式哈希：

`hash = sum(a[i] * BASE^i)`

当区间 `[l, r]` 内所有元素都加一时，哈希值增加：

`BASE^l + BASE^(l + 1) + ... + BASE^r`

所以只需要在线段树中维护区间哈希和，并支持区间加法即可。

真正麻烦的是取模。

当 `a[i] != 65535` 时：

`a[i] = a[i] + 1`

这是正常的加一。

但是当 `a[i] == 65535` 时：

`a[i] = 0`

此时如果我们仍然把它当作普通加一，它会从 `65535` 变成 `65536`。

正确值应该是 `0`，所以这个位置需要额外减去：

`65536`

对应到哈希上，就是额外减去：

`65536 * BASE^i`

因此一次区间加一可以拆成：

1. 找出区间内所有当前值为 `65535` 的位置；
2. 整个区间先统一加一；
3. 对这些溢出位置单独做哈希修正。

## 算法设计

代码中维护了两棵线段树。

第一棵线段树 `Seg` 维护当前数组值的最大值，用来判断区间内是否存在即将溢出的元素。

它支持：

* 区间加法；
* 单点赋值；
* 查询区间内第一个大于等于某个值的位置。

在本题中，用它来找区间 `[l, r]` 内所有值为 `65535` 的位置。

第二棵线段树 `HashSeg` 维护数组哈希值。

每个位置 `i` 的贡献为：

`a[i] * BASE^i`

所以区间 `[l, r]` 的哈希为：

`sum(a[i] * BASE^i), l <= i <= r`

为了快速支持区间加一，预处理：

`pw[i] = BASE^i`

以及：

`sum[i] = BASE^1 + BASE^2 + ... + BASE^i`

于是区间 `[l, r]` 整体加 `v` 时，哈希增加：

`v * (BASE^l + BASE^(l + 1) + ... + BASE^r)`

也就是：

`v * (sum[r] - sum[l - 1])`

## 操作一：区间加一

对于操作：

`1 l r`

代码先找出所有会溢出的位置：

```cpp
vector<int> over;

while (true) {
    int p = seg.find(l, r, LIM);
    if (p == -1) break;

    over.push_back(p);

    seg.setv(p, -1);
}
```

这里 `LIM = 65535`。

如果当前位置值是 `65535`，加一后应该变成 `0`。

代码先把这个位置在 `Seg` 里临时设为 `-1`，这样之后执行整段 `+1` 后，这个位置正好变成 `0`。

然后对整个区间执行普通加一：

```cpp
seg.add(l, r, 1);
hs.add(l, r, 1);
```

这一步中，哈希线段树也把整个区间当作普通加一处理。

但是对于原本为 `65535` 的位置，普通加一会把它看成 `65536`，而正确值应该是 `0`。

所以最后对这些位置单独修正：

```cpp
for (int p : over) {
    hs.add(p, p, -FULL);
}
```

其中：

`FULL = 65536`

这相当于从哈希中减去：

`65536 * BASE^p`

## 操作二：判断两个子数组是否相同

对于操作：

`2 x y len`

需要比较：

`a[x ... x + len - 1]`

和：

`a[y ... y + len - 1]`

代码先查询两个区间的哈希：

```cpp
ll h1 = hs.ask(x, x + len - 1);
ll h2 = hs.ask(y, y + len - 1);
```

由于哈希中位置权重是 `BASE^i`，两个相同的子数组如果起点不同，直接比较哈希是不行的。

例如同一个序列放在位置 `x` 和位置 `y`，它们的哈希会差一个 `BASE^(y - x)` 的倍数。

所以需要对齐。

如果 `x < y`，把第一个哈希右移到以 `y` 开头的位置：

```cpp
h1 = hs.shift(h1, y - x);
```

如果 `x > y`，则把第二个哈希右移：

```cpp
h2 = hs.shift(h2, x - y);
```

最后比较两个哈希是否相等：

```cpp
cout << (h1 == h2 ? "yes\n" : "no\n");
```

## 正确性说明

首先，证明哈希线段树可以正确维护普通区间加法。

哈希定义为：

`H = sum(a[i] * BASE^i)`

如果区间 `[l, r]` 内所有元素都增加 `v`，那么新的哈希变化量为：

`sum(v * BASE^i), l <= i <= r`

也就是：

`v * (BASE^l + BASE^(l + 1) + ... + BASE^r)`

代码中用 `sum[r] - sum[l - 1]` 表示这一段 `BASE` 幂次和，所以 `HashSeg::apply` 可以正确更新区间哈希。

其次，考虑模 `65536` 的影响。

对于不等于 `65535` 的元素，加一后就是普通的 `+1`，哈希线段树的区间加一是正确的。

对于等于 `65535` 的元素，正确变化是：

`65535 -> 0`

而普通加一会得到：

`65535 -> 65536`

两者相差：

`65536`

因此，对每个溢出位置，在整体加一之后额外减去 `65536 * BASE^i`，即可得到正确哈希。

代码中先用 `Seg` 找出所有当前值为 `65535` 的位置，并记录到 `over` 中。因此修正的位置恰好是所有发生溢出的位置，不会漏掉，也不会多算。

同时，`Seg` 中把这些位置先设为 `-1`，再对整个区间加一，使得它们最终变成 `0`，这与真实数组取模后的值一致。其他位置正常加一，也与真实数组一致。

所以两棵线段树在每次操作后都维护了正确的数组信息。

最后，查询两个子数组是否相同。

哈希中第 `i` 个位置的权重为 `BASE^i`。如果两个子数组内容相同但起点不同，那么它们的哈希相差一个整体的 `BASE` 幂次倍数。

代码根据 `x` 和 `y` 的大小，将较靠左的子数组哈希乘上对应的 `BASE` 幂，使两段哈希对齐到相同位置。对齐后，如果两个子数组完全相同，它们的哈希一定相同。

因此，算法可以正确回答所有查询。

## 复杂度分析

设发生溢出的总次数为 `C`。

每次操作的线段树修改或查询复杂度为：

`O(log n)`

对于每个溢出位置，也会进行一次查找和一次修正，复杂度为：

`O(log n)`

因此总时间复杂度为：

`O((q + C) log n)`

接下来估计 `C`。

一个元素每被加 `65536` 次，才会发生一次从 `65535` 到 `0` 的溢出。

所有区间操作对所有元素造成的总加法次数最多是：

`O(nq)`

所以溢出总次数最多约为：

`O(nq / 65536)`

因此整体复杂度可以写作：

`O((q + nq / 65536) log n)`

在本题数据范围内可以通过。

空间复杂度为：

`O(n)`

需要两棵线段树和若干预处理数组。

## 易错点

1. `65536` 和 `65535` 不要写混。

   判断溢出的位置是当前值等于 `65535`。

   哈希修正时减去的是 `65536`。

2. 哈希线段树一定要根据初始数组 `build`。

   如果只 `init` 不 `build`，初始哈希全是 `0`。

3. 哈希线段树区间递归修改后，一定要 `pull`。

   否则父节点哈希不会更新。

4. 处理负数取模时要使用 `norm`。

   因为溢出修正中会执行：

   `hs.add(p, p, -65536)`

5. 比较两个子数组哈希时，必须根据起点差异进行对齐。

   不能直接比较两个区间哈希。

6. 这份代码使用的是单模哈希，理论上存在哈希冲突。实际比赛中建议使用双模哈希或者 `unsigned long long` 进一步降低冲突概率。
