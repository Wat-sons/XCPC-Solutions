## 题意概述

给定 n 个操作，每个操作包含两个参数 t, v 。

我们需要在长度为  m  的序列上进行覆盖：

* 每个操作会以周期 2t 重复影响区间
* 每次影响一个长度为 t 的连续区间
* 被覆盖的位置会被赋值为对应的 v

不同操作之间可能覆盖同一位置，最终每个位置只保留**优先级更高（v 更大）的操作结果**。

输出最终序列。

## 解法二
### 关键观察

#### 1. 按价值从大到小处理是正确的

因为最终每个位置只会保留最大的 ( v )，所以我们可以：

> 先处理大的 v，再处理小的 v，避免重复覆盖。

---

#### 2. 每个 t 只需要处理一次

同一个 ( t ) 的操作如果重复出现，较小的 ( v ) 不可能影响最终答案，因此可以直接跳过。



#### 3. 并查集用于“跳过已填充位置”

我们维护一个 DSU：

* `find(x)`：找到 ≥ x 的第一个未被填的位置
* `merge(x)`：将 x 标记为已使用，并跳到 x+1

这样可以把原本可能 O(m) 的扫描优化为近似 O(α(m))。



#### 4. 结构化覆盖方式

对于每个操作 ( (t, v) )：

我们按周期处理：

```
j = 1, 1 + 2t, 1 + 4t, ...
```

每次覆盖区间：

```
[j, j + t - 1]
```

#### 算法设计

整体流程如下：

1. 按  v 从大到小排序所有操作
2. 用 set 记录已使用的 t（去重）
3. 对每个操作：

   * 枚举其周期起点
   * 在区间内用 DSU 跳跃填充未访问位置
4. 每个位置最多被赋值一次

### 正确性说明

#### 贪心正确性

由于按 v 降序处理：

* 先写入的位置一定是当前最优
* 后续较小 ( v ) 不可能覆盖已有更优答案

#### DSU 正确性

* `find(x)` 永远返回第一个未被覆盖的位置
* `merge(x)` 保证每个位置最多处理一次
* 因此不会重复写入，也不会遗漏

#### 周期枚举正确性

因为题目结构是周期性重复区间覆盖，所以按 2t 间隔枚举即可完整覆盖所有影响区域。

#### 复杂度分析

- 排序：O(n log n)
- DSU 操作：O(m α(m))
- 总复杂度：

> 时间复杂度：O(n log n + m α(m))  
> 空间复杂度：O(m)

* 忘记对 ( t ) 去重（同 t 只保留最大 v）
* 区间右端点要 `min(m, j+t-1)`
* DSU 必须路径压缩，否则会退化
* 注意 `find(pos)` 更新后要重新判断边界
