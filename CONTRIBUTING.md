# 贡献指南

## 提交内容范围

可以提交：

- AC 代码
- 题解 Markdown
- 补题链接与参考资料
- 题面 PDF（如仓库策略允许）
- 训练总结、坑点总结、模板代码

不建议提交：

- 明显未完成、无注释、无法复现思路的草稿
- 与题目无关的私有材料
- 可能违反题目平台规则或版权要求的完整原始题面内容（除非团队明确允许）

## 提交格式

建议每次提交只做一类事情：

- `new: 添加 A 题题解`
- `update: 更新 B 题题解`
- `chore: 更新失效链接`
- `docs: 更新 README 内容`

## Markdown 规范

完整题解建议包含以下部分：

1. 题意概述
2. 关键观察
3. 算法设计
4. 正确性说明
5. 复杂度分析
6. 代码
7. 易错点 / 反例
8. 外部参考链接

如果较为冗余，也可以省略部分内容。

## 代码规范

- 代码可独立编译
- 若代码依赖特定编译选项，请写在文件开头注释中

## 版权与授权确认

提交内容即表示你确认：

- 你对提交内容拥有合法授权，或已获得必要许可
- 你同意按仓库的[授权说明](docs/licensing.md)提交与分发
- 如你使用了他人内容，请明确标注来源
